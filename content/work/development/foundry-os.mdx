Foundry came with a hard brief: a developer operations platform that had grown feature by feature for four years, carrying 20k teams and no coherent interface language. The ask was not a redesign — it was an operating system for the product to grow into.

## The work

We started with an audit of every screen in production — 140 of them — and reduced the interface to 9 patterns. Those patterns became a token-driven component library, built in React and shipped behind a flag while the old UI kept running.

<Figure src="/images/work/foundry-os-patterns.svg" alt="Nine interface patterns distilled from 140 production screens" caption="140 SCREENS → 9 PATTERNS · THE REDUCTION THAT PAID FOR THE PROJECT" />

Migration ran page by page for a quarter. No big-bang launch, no frozen roadmap — the product team kept shipping the whole time, which was the point.

## What endured

- The component library is now the default way features ship; 92% of new surface area uses it unmodified.
- Interface-related support tickets dropped by a third in the first two months.
- The documentation site outlived my engagement — the team maintains it themselves, which is the outcome I optimize for.
